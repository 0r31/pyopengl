gl_PatchVerticesIn

contains the number of vertices in the current patch

Signature

gl_PatchVerticesIn( )->

Description

Available only in the tessellation control and evaluation languages,
gl_PatchVerticesIn
contains the number of vertices in the input being processed by the shader. A single tessellation control or evaluation shader can read patches of differing sizes, and so th value of
gl_PatchVertexIn
may differ between patches.

Version Support

gl_PatchVerticesIn
